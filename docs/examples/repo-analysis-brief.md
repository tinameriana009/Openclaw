# Example brief: new-repo onboarding analysis

## Goal

Understand a new repository quickly enough to:

- identify the main entrypoints
- map major subsystems
- find risky modification areas
- produce a short onboarding memo

## Good first request

```text
Analyze this repository for a new engineer. Summarize the top-level architecture, likely entrypoints, major subsystems, important config/data directories, and the highest-risk areas for modification. End with a suggested reading order.
```
